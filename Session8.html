Mengapa Membuat Komponen?

Reusable UI

Jika Anda membuat UI yang sama berulang kali (misal: Header khusus), lebih baik dijadikan satu komponen.

Keuntungan Utama:

  • Konsistensi: Tampilan seragam di seluruh aplikasi.
  • Efisiensi: Cukup drag-and-drop dari Palette.
  • Kapsulasi: Logika UI tersembunyi di dalam komponen.

Tipe Komponen Dasar

Di Delphi, kita biasanya menurunkan (Inheritance) dari class yang sudah ada:

TComponent

Non-visual komponen (seperti Timer atau Connection).

TControl / TWinControl

Komponen visual yang memiliki posisi, ukuran, dan bisa menerima input.

Frame (TFrame): Cara termudah untuk membuat "User Control" dengan menggabungkan beberapa komponen standar ke dalam satu wadah.

Visual Component: TFrame

Frame adalah cara "visual" membuat komponen tanpa menulis banyak kode manual:

  1. Pilih File > New > Other > Delphi Files > Frame.
  2. Letakkan Button, Edit, Label ke dalam Frame.
  3. Simpan Frame.
  4. Di Form utama, pilih Frames dari Standard Palette, lalu pilih Frame yang baru dibuat.

Frame bersifat sinkron; ubah desain Frame, maka semua Form yang menggunakannya akan ikut berubah.

Membuat Komponen Baru (Code)

Untuk membuat komponen yang muncul di Palette secara permanen:

procedure Register;
begin
  RegisterComponents('MyCustomPalette', [TMyCoolButton]);
end;
  • Gunakan Component > New VCL Component.
  • Pilih Class induk (misal: TButton).
  • Tambahkan Property baru (misal: ID_Karyawan).
  • Install Package untuk memasukkannya ke IDE.

Latihan Sesi 8

Challenge: Smart Input Frame

  1. Buat sebuah TFrame baru.
  2. Isi dengan satu TEdit dan satu TLabel (sebagai judul input).
  3. Buat Property di dalam Frame tersebut untuk mengubah Caption Label secara dinamis.
  4. Gunakan Frame ini sebanyak 3 kali di Form utama (Nama, Alamat, No Telepon).
