Mengapa Program Bisa Error?

Tipe Error

  • Syntax Error: Salah ketik kode (terdeteksi saat compile).
  • Runtime Error: Error saat aplikasi jalan (misal: bagi dengan nol).
  • Logic Error: Program jalan tapi hasilnya salah.

Aplikasi yang baik tidak akan "Crash", tetapi memberikan pesan informasi yang jelas kepada user.

Blok Try...Except

Gunakan blok ini untuk menangkap (catch) error agar aplikasi tetap berjalan.

try
  Hasil := Angka1 / Angka2;
except
  on E: Exception do
    ShowMessage('Terjadi kesalahan: ' + E.Message);
end;

Try: Jalankan kode yang berisiko.
Except: Kode cadangan jika terjadi masalah.

Blok Try...Finally

Digunakan untuk memastikan sumber daya (memory/file) tetap dibebaskan meskipun terjadi error.

MyQuery := TFDQuery.Create(nil);
try
  // Proses Query di sini
finally
  MyQuery.Free; // Ini AKAN SELALU dijalankan
end;

Penting untuk mencegah Memory Leak pada aplikasi skala besar.

Debugging di Delphi

Delphi menyediakan alat canggih untuk mencari sumber masalah:

Breakpoints (F5)

Memberhentikan baris kode yang sedang berjalan untuk diperiksa nilainya.

Watch List (Ctrl+Alt+W)

Memantau perubahan isi variabel secara real-time saat langkah demi langkah (Step Over F8).
