Apa itu OOP?

Pemrograman Berorientasi Objek

Paradigma pemrograman yang berfokus pada "Objek" yang memiliki data (atribut) dan perilaku (fungsi).

Tujuan Utama:

  • Reusability: Kode bisa digunakan kembali.
  • Maintainability: Kode lebih mudah dikelola.
  • Scalability: Aplikasi mudah dikembangkan.

Class vs Object

Class (Cetak Biru)

Definisi atau rancangan dari sesuatu. Contoh: Class TMobil.

Object (Implementasi)

Wujud nyata dari Class. Contoh: Mobil Avanza, Mobil Tesla.

Struktur Class

type
  TUser = class
  private
    FNama: string; // Fields (Private)
  public
    procedure Sapa; // Method
    property Nama: string read FNama write FNama; // Property
  end;
  • Private: Hanya bisa diakses oleh class itu sendiri.
  • Public: Bisa diakses dari luar class.
  • Property: Jembatan aman untuk mengakses field private.

Constructor & Destructor

Prosedur khusus untuk membuat dan menghapus objek dari memori.

// Membuat objek
MyUser := TUser.Create; 

// Menghapus objek (Freeing Memory)
MyUser.Free;

Encapsulation: Menyembunyikan detail implementasi dan hanya menampilkan fungsi penting kepada pengguna class.
