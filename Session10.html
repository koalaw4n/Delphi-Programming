Sesi 10: Multi-Threading Dasar & Respon UI

Oleh: Muhammad Ikhwan Fathulloh

Masalah: Aplikasi Hang / Freeze

Single Thread

Secara default, Delphi berjalan di satu alur (Main Thread). Jika ada proses lama, UI tidak bisa diklik (Not Responding).

Multi Thread

Menjalankan proses berat di "jalur belakang" (Background) sehingga UI tetap lancar dan bisa digunakan.

Cara Termudah: Anonymous Thread

Delphi menyediakan cara cepat menjalankan kode di background tanpa membuat file unit baru:

TThread.CreateAnonymousThread(
  procedure
  begin
    // Jalankan proses berat di sini (misal: Download file)
    Sleep(5000); 
  end
).Start;

Start memicu thread untuk langsung berjalan.

Aturan Emas: Sinkronisasi UI

DILARANG KERAS:

Mengakses komponen UI (seperti Label1.Caption atau Memo1.Lines) langsung dari dalam Background Thread. Ini akan menyebabkan Crash atau error aneh.

Solusi: TThread.Synchronize

TThread.Synchronize(nil,
  procedure
  begin
    Label1.Caption := 'Proses Selesai!';
  end);

Mengenal TTask

Untuk manajemen thread yang lebih modern, Delphi menggunakan TTask:

ITask

Representasi satu pekerjaan. Bisa ditunggu (Wait) atau dibatalkan (Cancel).

TParallel.For

Menjalankan perulangan (Looping) secara paralel menggunakan semua inti (Core) CPU Anda.

Latihan Sesi 10

Challenge: Responsiveness Test

  1. Buat tombol yang menjalankan perintah Sleep(5000) di Main Thread. Amati apa yang terjadi pada aplikasi.
  2. Pindahkan perintah tersebut ke dalam TThread.CreateAnonymousThread.
  3. Gunakan TThread.Synchronize untuk menampilkan pesan "Tugas Selesai" ke sebuah Label setelah 5 detik tersebut berakhir.
  4. Pastikan aplikasi tetap bisa digeser (drag) saat proses sedang berjalan.

Thread yang baik membuat User jatuh cinta pada kecepatan aplikasi Anda!
